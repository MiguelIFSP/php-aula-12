1. API

Uma API é um conjunto de regras e protocolos que permitem a comunicação entre diferentes softwares ou sistemas.

2. Exemplo

Se uma requisição é feita à: https://jsonplaceholder.typicode.com/posts/1

/posts
indica que queremos um post.
/1
indica o número do id do post.

O resultado é o seguinte JSON: 

{
  "userId": 1,
  "id": 1,
  "title": "sunt aut facere repellat provident occaecati excepturi optio reprehenderit",
  "body": "quia et suscipit\nsuscipit recusandae consequuntur expedita et cum\nreprehenderit molestiae ut ut quas totam\nnostrum rerum est autem sunt rem eveniet architecto"
}

É possível retornar XML, JSON ou mesmo texto simples. O mais recomendado é o formato JSON pela facilidade e quantidade de linguagens que possuem métodos embutidos para o trabalho com JSON.

Como ficaria uma página HTML/JS que consome o conteúdo da resposta: 

<!DOCTYPE html>
<html lang="pt-BR">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Acessa e mostra o resultado</title>
    <script>
	window.onload = inicio;

	function inicio() {
	    var botao = document.getElementById('buscar');
	    botao.addEventListener('click', busca_e_mostra);
	}

	function busca_e_mostra() {
	    const xhttp = new XMLHttpRequest();
	    xhttp.onload = function () {
		var resultado = this.responseText;
		var objeto = JSON.parse(resultado);
		var saida = document.getElementById('resposta');
		saida.innerHTML = objeto.title;
	    }
	    xhttp.open("GET", "https://jsonplaceholder.typicode.com/posts/1");
	    xhttp.send();
	}
    </script>
</head>

<body>
    <p id='resposta'></p>
    <form action="">
	<input type="button" value="Buscar" id='buscar'>
    </form>
</body>

</html>

3. Prática

A. Receba do usuário o número do post e mostre o title de acordo com o número selecionado.
